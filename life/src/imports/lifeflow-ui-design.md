Create a modern responsive healthcare website UI design for a platform called **“LifeFlow – AI Enabled Blood Donation & Request”**.

Use a clean **white and red medical theme**, modern healthcare UI style, soft shadows, rounded cards, and minimal layout similar to modern health tech platforms.

The website should include the following pages and UI sections:

1. **Landing Page (Home Page)**
   Hero section with headline “Save Lives with Smart Blood Donation”.
   Include illustration of blood donation.
   Two buttons: “Find Donors” and “Become a Donor”.
   Navigation bar with logo and menu items: Home, About, Donor, Patient, Login, Signup.
   Add sections explaining how the platform works, AI donor matching, emergency blood request, nearby hospitals, and footer with contact information.

2. **Role Selection Page**
   A simple centered layout where users choose their role.
   Two large cards:
   • Donor
   • Patient
   Each card should include an icon, short description, and select button.

3. **Login Page**
   Split layout page with illustration on the left and login form on the right.
   Fields:
   Email or phone number
   Password
   Login button
   Forgot password link
   Signup link.

4. **Signup Page**
   Clean signup form for new users.
   Fields include:
   Full Name
   Email
   Phone Number
   Password
   Confirm Password
   Role selection dropdown (Donor or Patient).

5. **OTP Verification Page**
   Verification screen with 6 OTP input boxes.
   Include verify button and resend OTP option.

6. **Donor Profile Setup Page**
   Form fields:
   Full name
   Blood group dropdown
   Age
   Gender
   Location
   Contact number
   Last donation date
   Save profile button.

7. **Donor Dashboard**
   Modern dashboard layout with top navigation bar including notification and settings icons.
   Main section includes a toggle button labeled **“Available to Donate Now”**.
   Dashboard cards include:
   Nearby Blood Requests
   Chat with Patient
   Check Requirements
   Donation History.

8. **Nearby Blood Requests Page**
   Display cards with patient request details including:
   Patient name
   Blood group needed
   Units required
   Hospital name
   Distance
   Accept request button.

9. **Donor Requirements Page**
   Informational page showing eligibility conditions for blood donation such as:
   Age requirements
   Weight requirement
   Health conditions
   Donation gap.
   Include button labeled **Check Eligibility**.

10. **Eligibility Check Page**
    Questionnaire style page with yes/no questions about health, surgery history, medication, and donation interval.
    Display result as **Eligible** or **Not Eligible**.

11. **Book Appointment Page**
    Appointment booking interface with:
    Date picker
    Time slot selection
    Hospital selection
    Confirm appointment button.

12. **Appointment Confirmation Page**
    Success message saying **Appointment Confirmed**.
    Display appointment details:
    Date
    Time
    Hospital name.
    Include buttons:
    Chat with Patient
    Back to Dashboard.

13. **Patient Dashboard**
    Dashboard layout showing:
    Location information
    Message “Searching for nearby donors”.
    Buttons for:
    View Map
    View Matches
    Nearby Donors
    Nearby Hospitals
    Emergency Blood Request.

14. **Matched Donors Page**
    Display donor cards showing:
    Donor name
    Blood group
    Distance
    AI match percentage
    Contact button.

15. **Nearby Hospitals Page**
    Hospital list cards including:
    Hospital name
    Distance
    Contact information.

16. **Emergency Blood Request Page**
    Form for critical blood requests including:
    Patient name
    Blood type
    Units required
    Hospital name
    Location details
    City or area
    Contact number.
    Include **Critical Emergency Mode toggle**.
    Button labeled **Notify Nearby Donors**.

17. **Nearby Donor Map Page**
    Interactive map showing nearby donors within 5 km radius and hospital locations.
    Markers should display donor information.

18. **Live Request Tracking Page**
    Request progress tracker showing stages:
    Request sent
    Donor matched
    Donor on the way
    Donation completed.

19. **Chat Page**
    Real-time chat interface between donor and patient.
    Include message bubbles, timestamps, and input box.

20. **Notifications Page**
    Notification list showing:
    Emergency alerts
    Donor matches
    Appointment reminders
    Chat messages.

21. **Settings Page**
    User settings dashboard including:
    Profile information
    Availability toggle
    Notification preferences
    Privacy settings
    Language
    Logout.

The design should be **modern, responsive, healthcare themed, and optimized for desktop website layout** with clean typography and medical icons.
